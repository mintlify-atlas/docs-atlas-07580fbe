---
title: "split() and splitBy() — Future fan-out API reference"
description: "API reference for split() and splitBy(). Fan out a single Future into two streams: errors vs values, or any custom predicate-based partition."
---

The split utilities let you fan out a single Future into two independent Futures that share the same underlying iterator. `split()` separates yielded values from thrown errors, giving you two clean channels to handle each case. `splitBy()` takes a predicate function and routes each value to one of two Futures depending on whether it matches. Both return a tuple that also implements `DualDisposable`, so you can use `await using` to clean up both channels together.

---

## `split()`

```typescript
function split<V, E, TReturn = unknown>(
  future: Future<V, TReturn>,
): readonly [Future<V, undefined>, Future<E, undefined>] & DualDisposable
```

Splits a Future into two Futures: one that receives resolved values and one that receives errors (thrown exceptions). Both Futures consume the same underlying async iterator, so each item is routed to exactly one of the two channels.

<ParamField path="future" type="Future<V, TReturn>" required>
  The Future to split. Its yielded values go to the first Future in the returned tuple; any thrown errors go to the second.
</ParamField>

<ResponseField name="return" type="readonly [Future<V, undefined>, Future<E, undefined>] & DualDisposable">
  A readonly two-element tuple where:

  <Expandable title="tuple elements">
    <ResponseField name="[0]" type="Future<V, undefined>">
      Receives all successfully resolved (yielded) values from the original Future. Its return type is `undefined` because the split pair does not propagate the original return value.
    </ResponseField>
    <ResponseField name="[1]" type="Future<E, undefined>">
      Receives all errors thrown by the original Future. The type parameter `E` must be specified explicitly — TypeScript cannot infer it from the source Future.
    </ResponseField>
  </Expandable>

  The tuple also implements `[Symbol.dispose]()` and `[Symbol.asyncDispose]()` from `DualDisposable`, so the entire split pair can be cleaned up with `using` or `await using`.
</ResponseField>

### Example

```typescript
import { from, split } from "@okikio/future";

const future = from(async function* () {
  yield 1;
  yield 2;
  throw new Error("An error occurred");
});

const [resolvedFuture, errorFuture] = split(future);

resolvedFuture.then(console.log); // Logs 1, 2
errorFuture.catch(console.error); // Logs Error: An error occurred
```

### Disposing the split pair

The return value is also a `DualDisposable`, which means you can use `await using` to automatically clean up both Futures when the enclosing block exits.

```typescript
import { from, split } from "@okikio/future";

{
  await using [values, errors] = split(future);
  // consume values and errors here
} // Both Futures are disposed automatically when this block exits
```

## `splitBy()`

```typescript
function splitBy<T, F, VReturn = unknown>(
  future: Future<T | F, VReturn>,
  predicate: (value: T | F) => boolean | PromiseLike<boolean>,
): readonly [Future<T, VReturn | undefined>, Future<F, VReturn | undefined>] & DualDisposable
```

Splits a Future into two Futures based on a predicate function. Each value yielded by the original Future is passed to `predicate`. Values for which `predicate` returns `true` go to the first Future; all others go to the second.

<ParamField path="future" type="Future<T | F, VReturn>" required>
  The Future whose output to partition. The union type `T | F` reflects that values of either shape may arrive before the predicate routes them.
</ParamField>

<ParamField path="predicate" type="(value: T | F) => boolean | PromiseLike<boolean>" required>
  A function that decides which Future each value goes to. It may be synchronous or async — if it returns a `PromiseLike<boolean>`, the routing waits for the promise to settle before forwarding the value.
</ParamField>

<ResponseField name="return" type="readonly [Future<T, VReturn | undefined>, Future<F, VReturn | undefined>] & DualDisposable">
  A readonly two-element tuple where:

  <Expandable title="tuple elements">
    <ResponseField name="[0]" type="Future<T, VReturn | undefined>">
      Receives values for which `predicate` returned `true`. Its return type is `VReturn | undefined` to account for cases where the split ends before the original Future's return value is propagated.
    </ResponseField>
    <ResponseField name="[1]" type="Future<F, VReturn | undefined>">
      Receives values for which `predicate` returned `false`.
    </ResponseField>
  </Expandable>

  Like `split()`, the tuple also implements `DualDisposable`.
</ResponseField>

### Example: even/odd partitioning

```typescript
import { from, splitBy } from "@okikio/future";

const future = from(async function* () {
  yield 1;
  yield 2;
  yield 3;
  yield 4;
});

const isEven = (value: number) => value % 2 === 0;
const [evenFuture, oddFuture] = splitBy(future, isEven);

evenFuture.then(console.log); // Logs 2, 4
oddFuture.then(console.log);  // Logs 1, 3
```

### Example: async predicate

```typescript
import { from, splitBy } from "@okikio/future";

const future = from(async function* () {
  yield { id: 1, type: "A" };
  yield { id: 2, type: "B" };
  yield { id: 3, type: "A" };
});

// Predicate can be async — e.g., checking a remote cache
const isTypeA = async (item: { id: number; type: string }) => {
  return item.type === "A";
};

const [typeA, typeB] = splitBy(future, isTypeA);

for await (const item of typeA) {
  console.log("Type A:", item); // { id: 1, type: "A" }, { id: 3, type: "A" }
}
```

<Note>
  Both `split()` and `splitBy()` share the same underlying iterator. Consuming one Future at a rate significantly different from the other may introduce backpressure. If both Futures need to be consumed concurrently, drive them in separate async tasks rather than awaiting them sequentially.
</Note>
