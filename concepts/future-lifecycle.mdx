---
title: "Future lifecycle: idle, running, paused, and cancelled"
description: "Learn how a Future moves through its lifecycle states — from idle to running, paused, cancelled, or completed — and how to control each transition."
---

A `Future` is not simply a wrapper around a promise — it is a stateful object with a well-defined lifecycle. Every `Future` begins in the `idle` state when it is constructed, and it moves through a predictable sequence of states as execution progresses, pauses, completes, or is cancelled. Understanding these states and the methods that trigger transitions between them is fundamental to using `@okikio/future` effectively.

## Lifecycle states

The `Status` object defines six possible states a `Future` can occupy. At any given moment, a `Future` is in exactly one of them.

| State | Value | Description |
|---|---|---|
| `Status.Idle` | `"idle"` | Created but not yet started |
| `Status.Running` | `"running"` | Actively executing the generator |
| `Status.Paused` | `"paused"` | Suspended at a `yield` point |
| `Status.Completed` | `"completed"` | Finished successfully |
| `Status.Cancelled` | `"cancelled"` | Aborted via `cancel()` |
| `Status.Destroyed` | `"destroyed"` | Disposed; cannot be reused |

### State transition flow

```
idle
 │
 ▼
running ──────────────────────────┐
 │                                │
 ▼                                │
paused ── resume() ──► running    │
                                  │
running ── complete() ──► completed ── reset() ──► idle
running ── cancel()  ──► cancelled ── reset() ──► idle
completed / cancelled ── dispose() ──► destroyed
```

<Note>
Once a `Future` reaches the `destroyed` state it cannot be reset or iterated. Calling `[Symbol.asyncIterator]()` on a destroyed `Future` throws an error.
</Note>

## Checking the current state

Two methods expose the current status of a `Future`:

```typescript
future.is(Status.Paused);   // true if currently paused
future.getStatus();          // returns the StatusEnum string, e.g. "paused"
```

`is()` accepts any member of `Status` and returns a `boolean`. `getStatus()` returns the raw `StatusEnum` string value, useful for logging or conditional branching.

```typescript
import { from, Status } from "@okikio/future";

const future = from(async function* () {
  yield 1;
  return 2;
});

console.log(future.getStatus());       // "idle"
console.log(future.is(Status.Idle));   // true
```

## Pausing and resuming

`pause()` suspends execution at the next `yield` point. The generator will not advance until `resume()` is called. Both methods return the `Future` instance for chaining.

```typescript
import { from } from "@okikio/future";

const future = from(async function* () {
  yield 1;
  yield 2;
  return 3;
});

future.pause();
// Execution is suspended; the generator will not advance past yield 1

setTimeout(() => future.resume(), 1000);
// Execution resumes after 1 second

for await (const value of future) {
  console.log(value); // Logs 1, 2 after the resume
}
```

<Tip>
`pause()` is a no-op if the `Future` is already paused. `resume()` is a no-op if the `Future` is not in the `paused` state.
</Tip>

## Completing a Future manually

`complete(value)` forces a `Future` into the `completed` state immediately, short-circuiting any remaining generator steps. The provided value becomes the resolved value.

```typescript
import { from, Status } from "@okikio/future";

const future = from(async function* () {
  yield 1;
  yield 2;
  return 3;
});

future.complete(99);
console.log(future.is(Status.Completed)); // true
```

<Warning>
Calling `complete()` while the generator is mid-execution causes `generator.return()` to be invoked internally, so any `finally` blocks in your generator will still run.
</Warning>

## Cancelling a Future

`cancel(reason?)` is an async method that aborts the underlying `AbortController` and waits for the `cancelled` status event to fire. It returns a `StatusEvent` (or `Error` if something went wrong during teardown).

```typescript
async cancel(reason: unknown = new CancellationError()): Promise<StatusEvent | Error | undefined>
```

When no reason is provided, the default is a `CancellationError` with the message `"Future was canceled"`.

```typescript
import { from } from "@okikio/future";

const future = from(async function* (abort) {
  for (let i = 0; i < 100; i++) {
    abort.signal.throwIfAborted(); // propagates cancellation into the loop
    yield i;
  }
});

setTimeout(() => future.cancel(), 500);

try {
  for await (const value of future) {
    console.log(value);
  }
} catch (err) {
  console.error("Cancelled:", err.message); // "Future was canceled"
}
```

<Steps>
  <Step title="cancel() is called">
    The method aborts the internal `AbortController` with the provided reason (defaulting to `CancellationError`).
  </Step>
  <Step title="The abort event fires">
    The `Future`'s internal abort listener sets the status to `cancelled` and dispatches a `StatusEvent`.
  </Step>
  <Step title="generator.return() is called">
    The wrapper calls `generator.return()` so that any `finally` blocks in your generator execute for cleanup.
  </Step>
  <Step title="cancel() resolves">
    The returned promise resolves with the `StatusEvent` for the `cancelled` state.
  </Step>
</Steps>

## Resetting a Future

`reset()` reinitialises a `Future` so it can be run again from the beginning. It can only be called after the `Future` is in the `completed` or `cancelled` state. Calling it on a running, paused, or destroyed `Future` throws an `Error`.

```typescript
import { from, Status } from "@okikio/future";

const future = from(async function* () {
  yield 1;
  return 2;
});

await future.toPromise();
console.log(future.is(Status.Completed)); // true

future.reset();
console.log(future.is(Status.Idle));      // true

// Run again from the start
console.log(await future.toPromise());    // 2
```

## Disposing a Future

`dispose()` performs a full teardown: it cancels the `Future`, disposes all registered resources on the internal `AsyncDisposableStack`, removes event listeners, and sets the status to `destroyed`. It is equivalent to calling `cancel()` followed by cleanup of every internal reference.

`[Symbol.asyncDispose]()` delegates directly to `dispose()`, enabling use with the `await using` syntax from the TC39 Explicit Resource Management proposal.

```typescript
import { from } from "@okikio/future";

{
  await using future = from(async function* () {
    yield 1;
    return 2;
  });

  // Use the future...
} // dispose() is called automatically here
```

## StatusEvent

A `StatusEvent` is the custom event object dispatched whenever a `Future` changes state. It extends `CustomEvent` and carries a `status` property identifying which state was entered, along with an optional `detail` payload.

```typescript
class StatusEvent<T extends StatusEnum, TDetail = unknown> extends CustomEvent<TDetail> {
  constructor(public status: T, detail?: TDetail)
}
```

For the `completed` state, `detail` contains `{ value: unknown }` — the resolved value. For the `cancelled` state, `detail` contains `{ reason: unknown }` — the abort reason. All other states carry no structured detail.

`cancel()` returns the `StatusEvent` for the `cancelled` transition, giving you direct access to the cancellation reason without subscribing to events manually.

```typescript
const event = await future.cancel(new Error("user clicked stop"));
console.log(event?.detail); // { reason: Error("user clicked stop") }
```
